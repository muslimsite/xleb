# Тёплый хлеб

Одностраничный сайт пекарни ремесленного хлеба на живой закваске (Москва).
Премиум-лендинг с рабочей корзиной, оформлением заказа и **отправкой заказов в Telegram**.

Сайт полностью статический — никакого сервера и сборки не нужно. Работает на GitHub Pages.

## Что умеет

- Каталог хлеба с добавлением в корзину (анимация «полёта» товара в корзину)
- Корзина: изменение количества, удаление, подсчёт суммы
- Конструктор «Собери свой хлеб»: мука, добавки, без глютена, размер, свои пожелания — попадает в корзину без цены (цена согласуется индивидуально)
- Оформление заказа: форма с именем, телефоном, способом получения (самовывоз / доставка такси) и комментарием
- Отправка собранного заказа в Telegram-чат (группу) одним сообщением
- Адаптив под мобильные, плавные анимации появления, уважение `prefers-reduced-motion`

## Структура

```
.
├── index.html    # разметка страницы

```

## Настройка Telegram (обязательно для приёма заказов)

Все шаги подробно описаны в комментариях внутри `config.js`. Кратко:

1. **Создать бота.** В Telegram откройте **@BotFather** → `/newbot` → получите **токен**
   вида `123456789:AAE...`.
2. **Создать группу** «Заказы — Тёплый хлеб», добавить в неё бота и пекаря(ей).
3. **Узнать chat_id группы.** Напишите в группе любое сообщение, затем откройте в браузере
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates` и найдите `"chat":{"id":-100...}`.
   Для группы id — **отрицательное число**.
4. Вставьте `BOT_TOKEN` и `CHAT_ID` в `config.js`, сохраните, закоммитьте.

Чтобы добавить или убрать пекаря — просто меняйте состав группы в Telegram, код трогать не нужно.

> ⚠️ Токен бота виден в коде сайта (так работает любой статический сайт без сервера).
> Это приемлемо для небольшого проекта. Если токен попадёт не в те руки — перевыпустите
> его у @BotFather (`/revoke`) и вставьте новый. Для полной защиты токена нужен прокси
> (Cloudflare Workers / Vercel) — это отдельная задача.

Пока токен не заполнен, кнопка «Отправить заказ» работает в **демо-режиме**:
заказ собирается, но в Telegram не уходит (об этом честно сообщается в окне).

## Локальный запуск

Можно открыть `index.html` двойным кликом. Если что-то не подгружается из-за `file://`,
поднимите локальный сервер:

```bash
python3 -m http.server 8000
# откройте http://localhost:8000
```

## Публикация на GitHub Pages

1. Создайте репозиторий на GitHub и запушьте эти файлы.
2. **Settings → Pages → Source: Deploy from a branch.**
3. Ветка `main`, папка `/ (root)`. Сохраните.
4. Через пару минут сайт будет доступен на `https://<username>.github.io/<repo>/`.

Файл `.nojekyll` уже добавлен, чтобы GitHub Pages отдавал файлы как есть.

## Редактирование ассортимента

Список хлеба и цены — в начале `app.js`, массив `ITEMS`. Меняйте название, цену,
вес, описание, тег и картинку (положите новое изображение в `assets/`).
